# The Query Profile

# Property Stereotypes which have a "hint" Attribute

For all these Stereotypes, the hint represents the default value, if you don't supply it, it means that the value is gonna be filled through a form:

# like

used to represent a like comparison on sql

# having

used to represent a having comparison on sql, must be used with apply_avg or apply_sum

# equalsTo

used to represent an equals comparison, can have a default value, and it's the default operation where no stereotype is applied

# greatherThan

comparison

# andGreatherThan

second comparison

# orGreatherThan

second comparison

# lessThan

comparison

# andLessThan

second comparison

# orLessThan

second comparison

# Property Stereotypes Applying special database functions

If the hint is set, it means select then ones which have "this function" with the value specified by the hint function by default these functions are intended to return value, except if a "hint" is set:

# apply_count

apply to input properties

# apply_sum

apply to input properties

# apply_avg

apply to input properties

# Property Stereotypes which don't have a "hint" Attribute

# defaultSystemId

# defaultUserId

default value

# isToday

default value

# isRightNow

default value

# isEmpty

comparison (check for nulls)

# isntEmpty
