# Profiles

In order to build the system by using the UML Language various Stereotypes must be used. A stereotype is like a second type to a class, so it can add meaning to it. For example: A "Client" in UML is a word generally used as reference to the entity "Client" from the real world, e.g., the person who buys products, and of course another kind of "Client"
