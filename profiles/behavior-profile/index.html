# The Behavior Profile

# Types

On our transformation the uml::Choice element is used to represent booth loops and decision points depending on the stereotypes being used.

  • If any kind of transition with name is leaving an uml::Choice element it means that a set of nested if statements are gonna be generated

  • If only numbers are used as name for the transitions leaving an uml::Choice it means that this transition will originate a "switch" statement

# Stereotypes

# True

If a Transition with this Stereotype is leaving a uml::Choice it means that an IF statement is going to be generated

# False

The same as Before, except by the fact that False has a different Meaning on the Program Flow

# Repeating

If a Repeating Transition is going to an uml::Choice it means that a while loop is going to be generated, in case a Repeating Transition is leaving a uml::Choice , it means tat a do-while loop is gonna be generated.

# DeclareVar

It's possible to declare a single variable on the program flow by using this Stereotype.

# Set

Used to set the value of a given variable.

# DeclareFuntion

Its used when Two or more Events footnoteref:[events, It can be a Button Event such as a Button Click or an Activity Result (For example after taking a picture)] will Generate the same Sequence of Actions so on the first state of those Sequences we apply the DelareFunction Stereotype.

# FunctionCall

Call to a function defined in an external Class. It's expected to be used with database classes (to ask questions about the database) and give the ability to change the Program Flow according to this data. Also to change the State of the system.

# Include

Includes another State Machine.

# MainBehavior

The Current State Machine represents the Behavior of the First Page (or Activity) of the Application.

  • The Behavior of the Items on the MainMenu if not specified by the respective State-Machines, will be represented here.
  • The Declaration of Variables and Types is possible By using External Classes

# Behavior

This Stereotype is similar to the MainBehavior Stereotype (also applied to uml State Machines ) but more generic.

  • In general it's used to declare variables used by a given State Machine
